## 一、项目概述

### 1\.1 项目背景

在软件研发与测试流程中，软件缺陷（Bug）是影响产品交付质量的核心要素。传统线下 Excel 记录缺陷、口头同步问题的管理方式存在缺陷丢失、流转无记录、追溯困难、测试用例与项目无法绑定、公告通知不及时等痛点，测试人员、开发、项目管理人员缺少统一线上协同管理载体。
本系统采用 SpringBoot\+Vue 前后端分离架构，搭建**软件缺陷跟踪管理平台**，区分普通测试用户与系统管理员两类角色，覆盖项目管理、测试用例管理、缺陷提交与跟踪、公告发布、留言反馈全流程，实现软件研发缺陷数字化闭环管理，完整记录缺陷提交、流转、处理全过程，提升软件测试与缺陷修复协同效率。

### 1\.2 开发技术栈

#### 后端技术（SpringBoot）

1. 核心框架：SpringBoot 2\.7\.x，标准化 RESTful 业务接口，分层解耦开发

2. 持久层框架：MyBatis\-Plus，实现多条件模糊检索、分页、批量新增 / 删除数据

3. 数据库：MySQL 8\.0，存储用户、项目分类、项目、用例分类、测试用例、缺陷信息、公告、留言反馈全量业务数据

4. 身份安全鉴权：JWT Token 无状态登录认证，区分普通用户、管理员角色，拦截未登录、越权访问接口

5. 安全机制：用户密码 MD5 加密存储，表单参数校验、重复提交拦截、登录会话时效控制

6. 文件上传模块：本地存储方案，支持缺陷截图、项目封面、测试用例图片上传预览

7. 工具组件：Hutool 工具库，封装日期处理、加密工具、分页工具、文件处理工具

#### 前端技术（Vue3 \+ Element Plus）

1. 核心框架：Vue3 \+ VueRouter \+ Pinia 全局状态管理

2. UI 组件库：Element Plus（管理员后台管理系统）、简约轻量化前台用户页面

3. 网络请求：Axios 全局请求 / 响应拦截器，自动携带 Token、捕获登录失效、统一异常提示

4. 页面通用组件：轮播图、图片上传、分类侧边筛选、卡片列表、分页表格、详情弹窗、多图预览

5. 交互功能：缺陷提交表单、公告点赞收藏、留言反馈、多维度数据检索筛选

### 1\.3 系统角色划分

1. **普通测试用户**
账号注册登录、浏览首页轮播与公告、查看项目信息（按项目分类筛选）、浏览测试用例、提交软件缺陷、查看个人提交缺陷、阅读公告、发布留言反馈、个人中心维护资料。

2. **系统管理员**
全后台管控权限：用户管理、项目类别 / 用例类别 / 缺陷类别字典维护、项目信息管理、测试用例管理、缺陷信息全量管理、留言反馈审核、平台公告发布、系统基础配置。

### 1\.4 整体业务流程

1. 注册登录：用户填写账号、姓名、密码、手机号完成注册；登录校验身份，区分跳转前台展示页或管理员后台

2. 测试人员前台流程：浏览项目列表→筛选对应项目→查看配套测试用例→提交项目缺陷（上传截图、填写问题描述）→浏览公告、提交留言反馈

3. 管理员后台流程：维护项目 / 用例 / 缺陷分类字典、录入项目与测试用例、统一管理所有缺陷单据、发布平台公告、处理用户留言反馈、管理平台全部注册用户

### 2\.1 登录注册模块

#### 2\.1\.1 登录页面

页面包含账号、密码输入框，登录按钮、注册用户跳转入口；后端校验账号密码密文，匹配成功生成 JWT Token，根据角色区分页面跳转，未登录状态无法访问系统核心功能。

#### 2\.1\.2 注册页面

必填字段：用户账号、姓名、密码、确认密码；选填：性别、手机号、头像上传；表单做前端非空、密码一致性校验，后端校验账号唯一性，完成注册入库。

### 2\.2 前台用户功能模块（顶部导航栏：首页、项目信息、测试用例、缺陷信息、公告栏、留言反馈）

1. 顶部全屏轮播横幅展示平台宣传图

2. 下方公告栏区域展示最新系统公告，提供「查看更多」入口跳转公告详情页

#### 2\.2\.2 项目信息页面

1. 左侧侧边栏：项目分类标签（全部、项目类别 1\~8），一键筛选对应分类项目

2. 顶部搜索框：输入项目名称模糊检索项目

#### 2\.2\.3 测试用例页面

1. 左侧侧边栏：用例分类筛选标签

2. 顶部搜索框：用例名称关键词检索

## 四、核心业务与安全流程

### 4\.1 登录鉴权安全流程

1. 用户输入账号密码提交登录接口，后端查询数据库比对 MD5 加密密码；

2. 校验通过生成 JWT Token，载荷存储用户 ID、角色类型、过期时间；

3. Token 存储至前端 localStorage，所有页面请求通过请求头自动携带 Token；

4. 后端全局拦截器校验：Token 为空 / 过期则强制跳转登录；角色无接口访问权限则返回 403 越权提示，同时记录访问日志；

5. 退出登录：前端清空本地 Token 缓存，下次访问需重新登录鉴权。

### 4\.2 软件缺陷提交跟踪闭环流程

1. 测试用户登录前台，浏览对应项目卡片；

2. 点击缺陷提交，填写问题名称、上传缺陷截图、描述问题详情提交；

3. 数据存入 bug\_info 缺陷表，标记初始状态「待处理」；

4. 管理员登录后台查看所有缺陷单据，分配处理人、标记缺陷修复进度、填写处理回复；

5. 测试人员前台个人中心查看本人缺陷的处理状态与管理员回复，完成缺陷闭环跟踪。

### 4\.3 后台基础数据运维流程

1. 管理员登录后台，先维护项目分类、用例分类、缺陷分类基础字典；

2. 录入项目信息、配套测试用例素材，绑定对应分类；

3. 日常管理用户提交的缺陷单据、处理留言反馈、发布平台公告；

4. 维护首页轮播图素材，配置前台首页展示内容。

## 五、项目部署说明

### 5\.1 SpringBoot 后端部署步骤

1. 本地 MySQL 执行项目提供的 SQL 初始化脚本，创建全部业务数据表，初始化默认管理员账号；

2. 修改`application.yml`核心配置项：

    - MySQL 数据库连接地址、账号、密码；

    - JWT 加密密钥、Token 有效过期时长；

    - 文件上传本地存储路径、文件大小限制；

    - 后端服务启动端口（默认 8080）、接口日志开关；

3. Maven 打包命令：`mvn clean package -DskipTests` 生成可执行 Jar 包；

4. Linux/Windows 服务器启动：`java -jar bug-trace-system.jar --server.port=8080`。

### 5\.2 Vue 前端部署步骤

1. 前端根目录执行`npm install`安装全部项目依赖；

2. 修改`axios`全局请求基地址，配置后端接口访问域名；

3. 生产环境打包：`npm run build`，生成 dist 静态资源文件夹；

4. Nginx 托管 dist 静态文件，配置反向代理转发 /api 请求至后端 8080 端口；

5. 区分前台用户访问路由、管理员后台管理路由，分开访问入口。

## 六、系统特色与创新亮点

### 6\.1 业务功能亮点（缺陷跟踪核心）

1. **完整缺陷生命周期管理**：从缺陷提交、分类标记、管理员处理、状态更新、结果回复形成完整闭环，所有操作永久存档可追溯；

2. 项目 \- 测试用例 \- 缺陷联动绑定：测试用例与项目关联，缺陷归属对应项目，多维度筛选检索，便于测试人员定位问题；

3. 图文化缺陷提交表单：支持多截图上传，完整留存缺陷现场截图证据，替代纯文字记录；

4. 前台轻量化交互页面：分类侧边筛选、卡片式展示项目 / 用例，操作简单，降低测试人员使用门槛；

5. 公告 \+ 留言反馈协同模块：平台统一发布测试规范、版本通知，用户线上留言沟通，减少线下沟通成本。

### 6\.2 系统安全设计亮点

1. **三级权限隔离**：普通测试用户仅可提交 / 查看本人缺陷，管理员拥有全量数据修改、删除权限，严格数据隔离；

2. 密码加密存储：数据库不保存明文密码，采用 MD5 不可逆加密，防止账号信息泄露；

3. Token 会话安全机制：登录凭证设置过期时间，退出主动销毁会话，避免会话劫持；

4. 多层参数校验：前端表单校验 \+ 后端接口参数双重校验，拦截非法、恶意提交数据；

5. 接口访问日志记录：后台可查看账号登录、数据修改操作记录，异常访问行为可审计追溯。

### 6\.3 拓展性优势

1. 分层架构低耦合：前端、后端、数据库完全分离，模块解耦，便于二次开发迭代；

2. 字典化分类管理：项目 / 用例 / 缺陷分类统一后台维护，无需修改代码即可新增分类；

3. 可拓展功能预留：支持后续新增缺陷导出 Excel、缺陷统计图表、消息推送、多角色细分（测试 / 开发 / 产品经理）等模块。

## 七、项目总结

本《基于 SpringBoot\+Vue 软件缺陷跟踪管理平台》面向软件测试业务场景开发，采用前后端分离架构，解决传统软件缺陷线下管理低效、无流程、难追溯的痛点。系统区分普通测试用户与管理员，实现项目管理、测试用例管理、缺陷全生命周期跟踪、公告发布、留言反馈一体化线上协同平台；配套完善的 JWT 鉴权、密码加密、接口权限拦截等安全机制，保障缺陷数据、用户账号安全。
项目代码分层清晰、业务流程贴合真实软件测试工作流，既满足计算机专业毕业设计、课程设计课题交付要求，也可直接迭代优化落地为中小企业内部测试缺陷管理工具，具备良好的实用性、可维护性与拓展开发空间。
